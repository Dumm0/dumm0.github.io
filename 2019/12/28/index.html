1. 配置环境

以下我会尽量用简短的语言描述, 请跟紧步骤:
安装的目的:
安装Hugo—> 生成一个文件夹(就是你的博客)
安装Git —> 把你的博客部署到GitHub
安装文本编辑器 —> 更方便编辑博客的配置文件

建立Github仓库

  • 详细请参考b站up: CodeSheep非常详细, 再次不再赘述

安装git:

  • 点我安装Git
    Git是当今使用最广泛的版本控制系统(可简单理解为代码仓库), 廖雪峰Git教程

安装Hugo:

这是Hugo的官方文档

  1. Windows10: 点我下载Hugo—>点击之后,看到页面v开头的版本号(这里以v0.62.0为例)–>下拉,找到 Assets 一栏—>点击下载 hugo_0.62.0_Windows-64bit.zip 压缩包

  2. 接着, 将解压包解压到你想保存的目录里(注意目录尽量不要有中文), 博主解压到了 “D:\Hugo” 中.

  3. 查看是否解压正确, 点开你解压的文件夹, 若里面有三个文件, 则安装成功: - hugo.exe - LICENSE - README.md

  4. 添加Hugo到环境变量中 - 点击桌面左下角windos关机键旁边的'搜索’, 输入 “环境变量”, 点击环境变量, 跳出列表 —> 列表有两个框, 找到 “用户变量” 的框, 点击 Path —> 编辑 —> 新建 —> 输入你解压的文件夹(博主的: D:\Hugo), 只需要把Hugo文件夹的路径输入即可.

  5. 再打开cmd (组合键: windows键+R呼出, 输入cmd回车), 进入命令行黑框后, 输入

    hugo version

    若得到:

    Hugo Static Site Generator v0.61.0-9B445B9D windows/amd64 BuildDate: 2019-12-11T08:29:13Z

    则安装成功, 若失败, 请 仔细阅读上面步骤, 特别注意文件路径, 还不行请留言博主

安装文本编辑器

这里以Sublime为例:
点我查看安装Sublime教程

2.创建博客(启动Hugo)

若找不到GitBash, 请win+R 输入cmd, 利用命令行完成

命令行输入:  cd 你的Hugo路径
博主的:      d:
             cd Hugo
            (博主的路径是 D:\Hugo 所以先d: 进入d盘目录, 然后在cd Hugo 进入Hugo文件夹下)

  1. 创建博客根目录:

    • 打开命令行cmd, 输入你的Hugo路径 具体看↑
    • 跳转成功之后, 输入 hugo new site Fuck(这里是你创建的文件名, 可自行取)
    • 然后在你的 Hugo路径下会有创建的文件 Fuck, 你可以自行移动到任意地方
    • 这个创建的文件夹, 就是你的博客根文件夹了

  2. 点开你的博客根目录文件 —> 鼠标右键 —> 选择GitBash Here

  3. 选择主题: Hugo的主题库
    这里用 tranquilpeak主题

    • 下载方法:
    1. 在3的bash中, cd来到themes文件夹下: 
          cd themes
    git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git tranquilpeak
    2. 若3.1方法中, clone的太慢, 也可以直接进入 这里 —> 单击绿色按钮'Clone or Download’ —> 选择Download zip —> 下载完成之后解压到 themes 目录下

  2. 博客本地预览

    • 先输入 hugo 重新渲染
    • 在输入 hugo server -D 生成本地预览
    • 浏览器地址栏输入GitBash中返回的结果, 一般是localhost:1313

如果到这里都OK, Congratulations!, 离成功仅差一步

  1. 部署到远端Github仓库

    • 将 config.toml 中的 baseUrl 改为自己仓库的地址: https://XXXXXX.github.io/
    • 在博客根目录下, 输入Hugo, 然后会在根目录下生成一个 public 文件夹, 部署到远端的文件就是这个Public
    • 输入你 GitHub的用户名 以及 电子邮箱: 
      git config --global user.name "xxxxx"
git config --global user.email "xxxxx@xx.com"
    • 将public部署到你的仓库中 
          # 进入public文件夹
    cd public

    # 初始化
    git init

    # 添加public中的所有文件
    git add .

    # 提交描述，XXXX中是你这次提交的描述，可以随便写/也可以不写这个
    git commit -m "XXXXXXXX"

    # 链接到远端，把XXXX替换成自己的内容，不要忘了还有后缀.git
    # 要是手贱输入错误, 就git remote rm origin 把原来的origin删掉
    # 再输入一遍正确的即可
    git remote add origin https://github.com/XXXXX/XXXXX.github.io.git


    git pull origin master

    # 注: 要是下一步不行, 返回拒绝访问的话
    git push -u origin master
    # 就输入: git push origin master -f(-f: force的意思, 强制把文件push到仓库中, 会把之前仓库中的版本删除, 下下策)

  2. 使用方法

    • 更新博客: 
          hugo # 生成博客配置
    cd public
    git add .
          
    git commit -m "XXXX"
    git pull origin master
    git push origin master
    # 要是提示origin的问题的话, 参考博客 **3.3 部署远端博客** 中的有关origin的注释
